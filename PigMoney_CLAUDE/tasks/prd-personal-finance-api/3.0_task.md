## Overview

Implement the full vertical slice for **Income**. Income depends on Account (FK), so Task 2.0 must be complete first. This resource introduces filtered pagination (`startDate`, `endDate`, `accountId`), requiring a custom repository interface with `GetFilteredAsync` and `CountFilteredAsync`.

<requirements>
- Income entity inherits BaseEntity; has Amount, Date, Description, AccountId (FK to Account)
- IIncomeRepository extends IRepository<Income> and adds GetFilteredAsync + CountFilteredAsync
- IncomeFilterParams DTO carries optional startDate, endDate, accountId filters
- IncomeService returns Result<PaginatedList<IncomeResponse>> on list queries
- All filters are optional — absence returns all records paginated
- FluentValidation: Amount > 0, Date required, AccountId > 0, Description required
- DELETE income has no blocking rule (no dependents)
- All .cs files must start with the required `//created by:` comment header
</requirements>
