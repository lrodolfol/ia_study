# Task 7.0: Unit Tests — Application Layer

<critical>Read the prd.md and techspec.md files in this folder; if you do not read these files, your task will be invalidated</critical>

## Overview

Write and run all **unit tests** for the Application layer: service classes and FluentValidation validators for all 6 resources. No real database is used — repositories are mocked with a mocking library (e.g., Moq or NSubstitute). This task consolidates any missing unit tests not yet written in tasks 2–5.

<requirements>
- Test project: `tests/Application.Tests/Application.Tests.csproj`
- All tests use mocked IRepository<T> — no real DB
- Every service method has at least one passing test and one failure test (Result.Failure path)
- Every validator has tests for: all required fields missing, boundary values (Amount = 0 fails, Amount = 0.01 passes), cross-field rules
- Tests follow AAA pattern (Arrange / Act / Assert)
- No Console.WriteLine in tests — use xUnit output helpers if needed
- All test files must start with the required `//created by:` comment header
</requirements>
