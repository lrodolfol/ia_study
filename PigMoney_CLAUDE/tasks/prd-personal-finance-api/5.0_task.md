## Overview

Implement the full vertical slice for **Budget**. Budget depends on Category (Task 2.0). It introduces date-range and categoryId filtering, plus a cross-field validation rule (`StartDate < EndDate`). No blocking rule on DELETE.

<requirements>
- Budget entity inherits BaseEntity; FK CategoryId; has StartDate, EndDate (DateTime), LimitAmount (decimal)
- IBudgetRepository extends IRepository<Budget> with GetFilteredAsync + CountFilteredAsync (categoryId, startDate, endDate)
- BudgetService returns Result<PaginatedList<BudgetResponse>> on list queries
- FluentValidation: LimitAmount > 0, StartDate required, EndDate required, EndDate > StartDate (cross-field rule)
- All .cs files must start with the required `//created by:` comment header
</requirements>

## Success Criteria

- GET `/api/v1/budgets?categoryId=1&startDate=2026-01-01&endDate=2026-12-31` returns filtered paginated list
- POST `/api/v1/budgets` with `EndDate < StartDate` returns 400 with validation error
- GET `/api/v1/budgets/{id}` returns 404 for unknown ID
- DELETE `/api/v1/budgets/{id}` returns 204 (no blocking rule)
- `dotnet build` passes with zero errors
