### MINOR Issues

**1. Falta teste de GetAllAsync com dados populados para AccountService e CategoryService**

- **Arquivo**: `src/pigMoney.Tests/Application/Services/AccountServiceTests.cs` (linha ~127)
- **Arquivo**: `src/pigMoney.Tests/Application/Services/CategoryServiceTests.cs` (linha ~124)
- **Descricao**: Ambos os servicos testam apenas `GetAllAsync` com lista vazia. Os outros servicos (Income, Expense, Budget) testam tanto o cenario vazio quanto o cenario com dados (`GetAllAsync_NoFilters_ShouldReturnAllPaginated`). Para consistencia e cobertura completa, Account e Category deveriam ter um teste equivalente com dados populados validando que os items sao mapeados corretamente.
- **Sugestao**:
```csharp
[Fact]
public async Task GetAllAsync_WithData_ShouldReturnPaginatedList()
{
    var accounts = new List<Account>
    {
        new() { Id = 1, Name = "Checking", Type = AccountType.Checking, Balance = 100m, CreatedAt = DateTime.UtcNow, UpdatedAt = DateTime.UtcNow }
    };
    _repositoryMock.Setup(r => r.GetPagedAsync(1, 10)).ReturnsAsync(accounts);
    _repositoryMock.Setup(r => r.CountAsync()).ReturnsAsync(1);

    var result = await _service.GetAllAsync(1, 10);

    Assert.True(result.IsSuccess);
    Assert.Single(result.Value!.Items);
    Assert.Equal(1, result.Value.TotalCount);
}
```

**2. UpdateAccountRequestValidatorTests nao testa boundary value (Balance = -0.01)**

- **Arquivo**: `src/pigMoney.Tests/Application/Validators/UpdateAccountRequestValidatorTests.cs`
- **Descricao**: O `CreateAccountRequestValidatorTests` inclui um teste `ShouldHaveError_WhenBalanceIsMinimalNegative` (Balance = -0.01m), mas o correspondente `UpdateAccountRequestValidatorTests` nao possui esse teste de boundary. O mesmo ocorre com `UpdateCategoryRequestValidatorTests` que nao testa Description com tamanho maximo excedido (enquanto o Create testa).
- **Sugestao**: Adicionar testes de boundary nos Update validators para manter paridade com os Create validators.

**3. Uso de `DateTime.UtcNow` diretamente nos testes pode causar flakiness**

- **Arquivos**: Todos os Service tests e Budget validator tests
- **Descricao**: A task exige que testes "Mock Date when time-dependent". Embora os testes atuais nao dependam de comparacao de datas exatas (usam apenas para setup), o uso de `DateTime.UtcNow` em multiplos locais sem controle pode causar problemas se a logica de negocios passar a depender de comparacao temporal. E uma boa pratica abstrair o relogio.
- **Sugestao**: Considerar a criacao de uma interface `IDateTimeProvider` ou usar uma variavel fixa `var now = new DateTime(2026, 1, 1, 0, 0, 0, DateTimeKind.Utc)` nos testes.

**4. Nomes de testes nao seguem o padrao "should" da convencao do projeto**

- **Arquivos**: Varios service tests
- **Descricao**: A task exige que nomes de testes comecem com "should". A maioria dos testes de servico usa o padrao `MethodName_Condition_ShouldExpectedResult` o que e aceitavel, mas o `CreateAsync_ShouldReturnSuccess` nao descreve a condicao. Padrao mais claro seria `CreateAsync_WhenRequestIsValid_ShouldReturnSuccess`.
- **Sugestao**: Ajustar para `CreateAsync_WhenRequestIsValid_ShouldReturnSuccess` nos testes de criacao de todos os servicos para manter consistencia.

**5. ExpenseService UpdateAsync nao testa cenario de Account ou Category nao encontrados**

- **Arquivo**: `src/pigMoney.Tests/Application/Services/ExpenseServiceTests.cs`
- **Descricao**: O `UpdateAsync_WhenFound_ShouldReturnSuccess` configura mocks para Account e Category existirem, mas nao ha testes para os cenarios onde Account ou Category nao existem durante um Update (similares aos testes que existem no `CreateAsync`).
- **Sugestao**: Adicionar `UpdateAsync_WhenAccountNotFound_ShouldReturnFailure` e `UpdateAsync_WhenCategoryNotFound_ShouldReturnFailure`.
