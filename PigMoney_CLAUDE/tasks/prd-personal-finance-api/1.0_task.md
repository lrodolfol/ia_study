# Task 1.0: Foundation & Project Setup

<critical>Read the prd.md and techspec.md files in this folder; if you do not read these files, your task will be invalidated</critical>

## Overview

Set up the entire project foundation: solution structure, shared base types in Domain, EF Core + Npgsql wiring in Repository, and a functional Web API baseline in API. After this task the solution must compile and `dotnet run` must start the API with no errors.

<requirements>
- Solution file `pigMoney.slnx` must reference all 4 projects: Domain, Application, Repository, API
- `BaseEntity` with `Id (int)`, `CreatedAt (DateTime)`, `UpdatedAt (DateTime)` in Domain
- `Result<T>` record in `Domain/Common/Result.cs` following the spec
- Generic `IRepository<T>` interface in `Domain/Interfaces/IRepository.cs`
- `AppDbContext` in `Repository/Data/AppDbContext.cs` with Npgsql provider wired
- `Program.cs` replaced with full Web API pipeline: `AddControllers`, `MapControllers`, DI stubs ready
- EF Core + Npgsql packages added to Repository.csproj
- FluentValidation package added to Application.csproj
- Swashbuckle (Swagger) added to API.csproj
- Connection string in `appsettings.json` only — never in `launchSettings.json`
- All .cs files must start with the required `//created by:` comment header
</requirements>
