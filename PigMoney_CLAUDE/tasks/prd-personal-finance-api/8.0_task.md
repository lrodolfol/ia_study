# Task 8.0: Integration Tests — API Layer

<critical>Read the prd.md and techspec.md files in this folder; if you do not read these files, your task will be invalidated</critical>

## Overview

Write and run all **integration tests** for the API layer using `WebApplicationFactory<Program>` with a real PostgreSQL test database. These tests exercise the full HTTP request → response pipeline including middleware, validation, and the standard response envelope. Tasks 2–6 must all be complete before this task starts.

<requirements>
- Test project: `tests/API.Tests/API.Tests.csproj`
- Uses `WebApplicationFactory<Program>` — real HTTP pipeline, real DB (separate test connection string via environment variable)
- Every resource (6 total) has tests for: GET list (pagination fields present), GET by id, POST 201 with location header, PUT 200, DELETE 204 or 409
- All responses validated for standard envelope shape: `statusCode`, `message`, `error`, `data`
- 400 tests: POST with missing/invalid body → error array populated
- 404 tests: GET/PUT/DELETE with unknown ID
- 409 tests: DELETE Account (with incomes/expenses), DELETE Category (with expenses/budgets), DELETE Expense (with items)
- Pagination tests: verify TotalCount, Page, PageSize on list responses
- All test files must start with the required `//created by:` comment header
</requirements>
