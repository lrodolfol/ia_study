# Task 4.0: Expense & ExpenseItem Resources

<critical>Read the prd.md and techspec.md files in this folder; if you do not read these files, your task will be invalidated</critical>

## Overview

Implement the full vertical slice for **Expense** and **ExpenseItem**. Expense depends on both Account and Category (Task 2.0). ExpenseItem depends on Expense. Both are delivered together because ExpenseItem's nested routes live under the Expense controller. Expense introduces multi-param filtering (date, accountId, categoryId); DELETE Expense is blocked (409) if it has items.

<requirements>
- Expense entity inherits BaseEntity; FKs: AccountId, CategoryId; navigation list Items initialized in declaration
- ExpenseItem entity inherits BaseEntity; FK ExpenseId; has Name, Quantity (decimal), UnitPrice (decimal)
- IExpenseRepository extends IRepository<Expense> with GetFilteredAsync + CountFilteredAsync (date, accountId, categoryId)
- IExpenseItemRepository extends IRepository<ExpenseItem> with GetByExpenseIdAsync + CountByExpenseIdAsync
- DELETE Expense returns 409 if expense has any items
- Nested routes for items: POST/GET `/api/v1/expenses/{expenseId}/items`; standalone GET/PUT/DELETE `/api/v1/expense-items/{id}`
- FluentValidation for all 4 request types (CreateExpense, UpdateExpense, CreateExpenseItem, UpdateExpenseItem)
- All .cs files must start with the required `//created by:` comment header
</requirements>

## Success Criteria

- GET `/api/v1/expenses?page=1&pageSize=10&categoryId=1` returns filtered paginated list
- POST `/api/v1/expenses/{expenseId}/items` adds item and returns 201
- GET `/api/v1/expenses/{expenseId}/items` returns paginated items for that expense
- GET `/api/v1/expense-items/{id}` returns 404 for unknown ID
- DELETE `/api/v1/expenses/{id}` returns 409 when expense has items
- `dotnet build` passes with zero errors
