## Overview

Create the application layer's data transfer objects (DTOs) and validation logic. Implement Request and Response DTOs organized by entity type, create FluentValidation validators for all request DTOs, implement the PaginatedList<T> wrapper for list responses, and create filter parameter classes for querying expenses and incomes.

- Create Request DTOs (Create and Update) for all 6 entities

- PaginatedList<T> with Items, TotalCount, Page, PageSize, TotalPages

## Implementation Details

Reference techspec-personal-finance-api.md sections:
- **Request/Response DTOs** (lines 266-323): Complete DTO definitions with properties
- **System Architecture - Application Layer** (lines 29-35): Application layer structure
- **Integration Points - FluentValidation** (lines 490-494): Validation configuration

Key implementation points:
- Request DTOs use record types: `public record CreateExpenseRequest(decimal Amount, DateTime Date, ...)`
- Update DTOs have nullable properties for partial updates
- Response DTOs include related entity names (CategoryName, AccountName)
- Validators: RuleFor(x => x.Amount).GreaterThanOrEqualTo(0).WithMessage("Amount must be >= 0")
- Budget validator: RuleFor(x => x.EndDate).GreaterThan(x => x.StartDate)
- FilterParams include pagination: Page (default 1), PageSize (default 50)
- PaginatedList calculates TotalPages = (int)Math.Ceiling(TotalCount / (double)PageSize)
