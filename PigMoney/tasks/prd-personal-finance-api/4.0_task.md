## Overview

Implement business logic services for Category and Account entities. Create service interfaces and implementations with CRUD operations, soft delete validation logic (prevent deletion if entities have active transactions), DTO mapping, and integration with repository layer. These services form the foundation for transaction management.

<requirements>
- Implement ICategoryService and CategoryService with CRUD operations
- Implement IAccountService and AccountService with CRUD operations
- Soft delete validation: prevent deletion if category has active expenses/incomes/budgets
- Soft delete validation: prevent deletion if account has active expenses/incomes
- Map entities to Response DTOs with proper property mapping
- Return Result<T> for all operations (never throw exceptions for business logic)
- Use dependency injection for repository dependencies
- Follow SOLID principles and RULES.md standards
- Services in /src/Application/Services/ folder
- Pagination support for GetAll operations (default 50 items per page)
</requirements>

## Implementation Details

Reference techspec-personal-finance-api.md sections:
- **Main Interfaces - ICategoryService** (lines 99-106): Category service interface
- **Main Interfaces - IAccountService** (lines 109-116): Account service interface
- **System Architecture - Application Layer** (lines 29-35): Service layer responsibilities
- **Testing Approach - Critical Test Scenarios** (lines 521-527): Soft delete validation scenarios

Key implementation points:
- CategoryService constructor: `public CategoryService(IRepository<Category> repository, ILogger<CategoryService> logger)`
- Soft delete check for Category: Query expenses/incomes/budgets where CategoryId matches and IsDeleted=false
- Soft delete check for Account: Query expenses/incomes where AccountId matches and IsDeleted=false
- DTO mapping: `new CategoryResponse(entity.Id, entity.Name, entity.Type, entity.CreatedAt, entity.UpdatedAt)`
- Return Result.Failure("Cannot delete category with active transactions") when validation fails
- Pagination: `repository.GetAllAsync(page, pageSize)` then wrap in PaginatedList
- Update operations: Fetch entity, check if null, update properties, call repository.UpdateAsync
- Use structured logging: `_logger.LogInformation("Creating category with name {Name}", request.Name)`
