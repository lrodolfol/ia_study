Reference techspec-personal-finance-api.md sections:

- **Performance Requirements** (lines 218-222): Response time and concurrency targets

Key implementation points:
- Use WebApplicationFactory for E2E tests with test database
- Test data seeding:
  ```csharp
  var testCategory = new Category { Name = "Groceries", Type = TransactionType.Expense };
  var testAccount = new Account { Name = "Checking", Type = AccountType.Checking, InitialBalance = 1000 };
  await SeedDataAsync(testCategory, testAccount);
  ```
- E2E test structure: Setup → Act → Assert → Cleanup
- Performance test: Measure response times with Stopwatch or BenchmarkDotNet
- Concurrency test: Use Task.WhenAll with multiple HttpClient requests
- Data integrity tests: Verify error messages and database state after failed operations
- Load test: Create 1000+ transactions, measure query times for filtering operations
- Validation scenarios:
  - Expense items sum = parent amount (success)
  - Expense items sum > parent amount (failure)
  - Delete category with expenses (failure)
  - Delete category without expenses (success)
