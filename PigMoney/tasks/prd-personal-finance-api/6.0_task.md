# Task 6.0: Budget & ExpenseItem Services

<critical>Read the prd-personal-finance-api.md and techspec-personal-finance-api.md files in this folder; if you do not read these files, your task will be invalidated</critical>

## Overview

Implement business logic services for Budget and ExpenseItem entities. Create service interfaces and implementations with CRUD operations, category-based filtering for budgets, expense item sum validation (application-level check before database trigger), DTO mapping, and integration with repository layer. ExpenseItem validation is critical to prevent data integrity violations.

<requirements>
- Implement IBudgetService and BudgetService with CRUD operations
- Implement IExpenseItemService and ExpenseItemService with CRUD operations
- Budget filtering: GetBudgetsByCategoryAsync for category-specific budgets
- ExpenseItem sum validation: Total of expense items cannot exceed parent expense amount
- Validate CategoryId exists before creating budgets
- Validate ExpenseId exists before creating expense items
- Return Result<T> for all operations
- Use dependency injection for repository dependencies
- Follow SOLID principles and RULES.md standards
- Application-level validation complements database trigger for expense items
</requirements>

Key implementation points:
- BudgetService constructor: `public BudgetService(IRepository<Budget> repository, IRepository<Category> categoryRepository, ILogger<BudgetService> logger)`
- ExpenseItemService constructor: Inject IRepository<ExpenseItem>, IRepository<Expense>, ILogger
- Sum validation logic:
  ```csharp
  var parentExpense = await _expenseRepository.GetByIdAsync(request.ExpenseId);
  var existingItems = await _expenseItemRepository.GetByExpenseIdAsync(request.ExpenseId);
  var currentSum = existingItems.Sum(item => item.Amount);
  if (currentSum + request.Amount > parentExpense.Amount)
      return Result.Failure("Sum of expense items cannot exceed parent expense amount");
  ```
- GetBudgetsByCategoryAsync: Filter budgets where CategoryId matches parameter
- DTO mapping: Include all entity properties in response DTOs
- Update operations: Recalculate sum for expense items when updating amounts
