## Implementation Details

Reference techspec-personal-finance-api.md sections:
- **API Endpoints** (lines 430-472): Complete endpoint definitions with HTTP methods and status codes
- **System Architecture - API Layer** (lines 37-43): Route classes in /routes folder

Key implementation points:
- Route extension method pattern:
  ```csharp
  public static class ExpenseRoutes
  {
      public static void MapExpenseRoutes(this IEndpointRouteBuilder app)
      {
          app.MapPost("/api/v1/expenses", async (CreateExpenseRequest request, IExpenseService service) =>
          {
              var result = await service.CreateExpenseAsync(request);
              return result.IsSuccess 
                  ? Results.Created($"/api/v1/expenses/{result.Value.Id}", result.Value)
                  : Results.BadRequest(result.Error);
          });
          
          app.MapGet("/api/v1/expenses/{id}", async (int id, IExpenseService service) =>
          {
              var result = await service.GetExpenseByIdAsync(id);
              return result.IsSuccess ? Results.Ok(result.Value) : Results.NotFound(result.Error);
          });
      }
  }
  ```
- Query parameters: Use [AsParameters] attribute or method parameters for filtering
- Status code mapping:
  - 200: OK for GET/PUT/DELETE success
  - 201: Created for POST success with Location header
  - 400: BadRequest for validation failures
  - 404: NotFound for entity not found
  - 500: Handled by ExceptionHandlerMiddleware
- Register routes in Program.cs: `app.MapExpenseRoutes(); app.MapIncomeRoutes(); ...`
- Middleware handles response wrapping, so routes return Result<T> directly
