# Task 1.0: Foundation - Domain Layer & Base Infrastructure

<critical>Read the prd-personal-finance-api.md and techspec-personal-finance-api.md files in this folder; if you do not read these files, your task will be invalidated</critical>

## Overview

Establish the foundational domain layer by creating the BaseEntity abstract class, IAggregateRoot marker interface, Result<T> pattern for error handling, all domain entities (Expense, Income, Category, Budget, Account, ExpenseItem), and enums (TransactionType, AccountType). This task creates the core building blocks that all other layers will depend on.

<requirements>
- All entities must inherit from BaseEntity with int Id, CreatedAt, UpdatedAt, and IsDeleted properties
- Implement custom Result<T> and Result classes in Domain layer (no external libraries)
- All entities must be marked as aggregate roots via IAggregateRoot interface
- String properties must default to string.Empty (never null)
- List properties must be instantiated at declaration with []
- Follow SOLID principles and coding standards from RULES.md
- All entities must include navigation properties for EF Core relationships
- Enums must use explicit integer values starting from 0
</requirements>
