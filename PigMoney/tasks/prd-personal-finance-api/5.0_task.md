## Overview

Implement business logic services for Expense and Income entities. Create service interfaces and implementations with CRUD operations, advanced filtering capabilities (date range, category, account), pagination, DTO mapping with related entity names (CategoryName, AccountName), and integration with repository layer. These services handle the core transaction management functionality.

<requirements>
- Implement IExpenseService and ExpenseService with CRUD operations
- Implement IIncomeService and IncomeService with CRUD operations
- Advanced filtering: date range (StartDate, EndDate), CategoryId, AccountId
- Response DTOs include CategoryName and AccountName (not just IDs)
- Pagination for filtered results (default 50 items per page)
- Map entities to Response DTOs with related entity information
- Return Result<T> for all operations
- Validate CategoryId and AccountId exist before creating/updating
- Use dependency injection for repository dependencies
- Follow SOLID principles and RULES.md standards
</requirements>

## Implementation Details

Reference techspec-personal-finance-api.md sections:
- **Main Interfaces - IExpenseService** (lines 81-88): Expense service interface
- **Main Interfaces - IIncomeService** (lines 90-97): Income service interface
- **Request/Response DTOs** (lines 268-308): ExpenseResponse includes CategoryName and AccountName
- **Testing Approach - Critical Test Scenarios** (lines 521-527): Filtering and validation scenarios

Key implementation points:
- Service constructors: Inject IRepository<Expense>, IRepository<Category>, IRepository<Account>, ILogger
- CategoryId/AccountId validation: Check existence in respective repositories before creating
- Response DTO mapping: `new ExpenseResponse(Id, Amount, Date, CategoryId, Category.Name, AccountId, Account.Name, ...)`
- Filtering logic: Build LINQ query with Where clauses for each filter parameter
  - `query = query.Where(e => e.Date >= filters.StartDate)` when StartDate is not null
  - `query = query.Where(e => e.CategoryId == filters.CategoryId)` when CategoryId is not null
- Eager loading: Include Category and Account when fetching expenses/incomes
- Return Result.Failure("Category not found") when validation fails
- Pagination after filtering: Apply Skip/Take, calculate TotalCount before pagination
